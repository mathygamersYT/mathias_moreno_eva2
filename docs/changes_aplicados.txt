Cambios aplicados al proyecto IT Progs
-------------------------------------

1) Estructura semantica principal
- Se usa <header> para la cabecera global del sitio.
- Se usa <nav> con <ul> y <li> para los enlaces de navegacion.
- Se usa un unico <main> para el contenido principal.
- Cada bloque importante esta organizado como <section>.
- Los servicios destacados usan <article>, porque cada tarjeta funciona como una unidad de contenido independiente.
- El pie de pagina esta contenido en <footer>.

2) Accesibilidad
- Las imagenes tienen textos alt descriptivos y coherentes con la marca IT Progs.
- El formulario mantiene labels asociados a sus campos mediante los atributos for e id.
- Las secciones enlazadas desde el menu tienen tabindex="-1" para mejorar el foco al usar scroll suave.
- Los enlaces externos usan target="_blank" junto con rel="noopener".

3) Consistencia visual y responsive
- La marca se unifico como "IT Progs" en titulo, encabezados, textos alternativos y footer.
- El color principal se ajusto para mejorar contraste en botones y elementos destacados.
- La tabla de enlaces rapidos se envolvio en un contenedor con overflow horizontal para evitar cortes en pantallas pequenas.

4) Organizacion de archivos
- index.html contiene la estructura del sitio.
- css/styles.css contiene los estilos y reglas responsive.
- js/scripts.js contiene el scroll suave y el comportamiento del formulario.
- images/ contiene los SVG usados por la landing page.
- docs/ contiene documentacion y registro de apoyo del proyecto.
